Demo third-party app

Shows the three ways an app integrates with the Pryv user-account app: sign-in hand-off (learn who/where the user is), account-page links (self-service with a way back), and the access-request flow (obtain your own token). Serve this file from any web server and open it — no build step.

Configuration
1 — Sign-in hand-off (returnURL + state)

Sends the user to /signin; on success they come back here with state (CSRF check) and pryvApiEndpoint (no token — run the access-request flow below to get one). A returning user sees "Continue as {username}" instead of the form.

2 — Account pages (backUrl / backLabel)

Self-service pages with a "← Back to {this app}" link in the header.

3 — Access request (get a token, lib-js)

Standard Pryv access-request: the popup opens the platform's consent app; the promise resolves with an api endpoint carrying this app's own token.
